# Voice

Who's talking: a senior company officer, not a SaaS app. Terse, direct,
assumes the reader is a competent professional doing a knowledge check —
not a student who needs hand-holding.

## Rules

1. **No cheerleading.** No "Great job!", no exclamation points. A captain
   doesn't gush.
2. **No hedging filler.** Cut "This will help you...", "Feel free to...",
   "You can now...". State the thing.
3. **Don't over-explain the obvious.** These are pros. Trust the reader.
4. **No marketing rhythm.** Kill balanced triads ("faster, smarter,
   better") and empty encouragement ("You're doing amazing!"). Say
   something true instead, or say nothing.
5. **Real job language over abstractions.** "The call," not "the
   situation." "Lock it in," not "finalize your submission."

## Do / Don't

| Don't | Do |
|---|---|
| "Great job! Submit Run — Reveal Official Answers" | "See the answers" |
| "This run isn't saved. Create a free account and future solo runs land in your library." | "Not logged in — this run won't be saved." |
| "Answer every question first — no partial reveals!" | "Answer everything first." |
| "Awesome, you're all set!" | (say nothing, or state what happened: "Saved.") |

## Process

Every PR that touches user-facing copy writes to this file if it
establishes a new pattern. Redline against your own ear before shipping —
if it reads like generic AI-assistant copy, it's wrong for this app.
